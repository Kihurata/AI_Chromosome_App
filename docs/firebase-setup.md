# Firebase Setup Plan - Chromosome App

## Overview
Detailed plan to set up a new Firebase project from scratch, including integrating it into both the Flutter frontend (for Authentication, Firestore, and Storage using `flutterfire_cli`) and the FastAPI backend (using the Firebase Admin SDK).

## Project Type
**DESKTOP & MOBILE** (Flutter) + **BACKEND** (FastAPI)

## Success Criteria
- Firebase project is successfully created in the console with core services enabled.
- Flutter frontend initializes Firebase seamlessly across macOS/iOS/Android/Web.
- FastAPI backend securely initializes the Firebase Admin SDK using a service account key without leaking credentials.

## Tech Stack
- **Frontend Configuration:** `flutterfire_cli` and `firebase-tools` (enables cross-platform configuration automatically).
- **Frontend SDKs:** `firebase_core`, `firebase_auth`, `cloud_firestore`, `firebase_storage`.
- **Backend SDK:** `firebase-admin` (Python).

## File Structure Updates
```text
/
├── frontend/
│   ├── firebase.json              # Generated by FlutterFire CLI
│   ├── lib/
│   │   ├── firebase_options.dart  # Generated by FlutterFire CLI
│   │   └── main.dart              # Updated to initialize Firebase
│   └── pubspec.yaml               # Add Firebase dependencies
├── backend/
│   ├── app/
│   │   └── core/
│   │       └── firebase.py        # Firebase Admin SDK initialization
│   ├── serviceAccountKey.json     # Secret key (MUST BE GITIGNORED)
│   └── requirements.txt           # Add firebase-admin
└── .gitignore                     # Updated to ignore serviceAccountKey.json
```

## Task Breakdown

### Phase 1: Firebase Project Creation (Manual/Guided)

#### Task 1: Create Firebase Project & Enable Services
**Agent:** `orchestrator` | **Skill:** `behavioral-modes`
- **Dependencies:** None
- **INPUT:** Setup guidelines for Firebase Console.
- **OUTPUT:** Instructions to create the Firebase project, enable Authentication (Email/Google), Firestore Database (Native mode), and Storage.
- **VERIFY:** User confirms project exists in Firebase Console and specific services are active.

### Phase 2: Frontend Setup (Flutter)

#### Task 2: Install CLI Tools & Generate Config
**Agent:** `mobile-developer` | **Skill:** `mobile-design`
- **Dependencies:** Task 1
- **INPUT:** Terminal and existing `frontend` directory.
- **OUTPUT:** Execution of global CLI tools (`npm install -g firebase-tools`, `dart pub global activate flutterfire_cli`) and running `flutterfire configure`.
- **VERIFY:** `firebase_options.dart` and `firebase.json` map configurations cleanly without error.

#### Task 3: Initialize Firebase in Flutter App
**Agent:** `mobile-developer` | **Skill:** `mobile-design`
- **Dependencies:** Task 2
- **INPUT:** `frontend/lib/main.dart` & `frontend/pubspec.yaml`.
- **OUTPUT:** Added dependencies (`firebase_core`, `firebase_auth`, `cloud_firestore`, `firebase_storage`). Refactored `main.dart` with `await Firebase.initializeApp(...)`.
- **VERIFY:** Flutter app builds and runs successfully locally without initialization exceptions.

### Phase 3: Backend Setup (FastAPI)

#### Task 4: Secure Service Account Key
**Agent:** `backend-specialist` | **Skill:** `api-patterns`
- **Dependencies:** Task 1
- **INPUT:** Service Account key downloaded from Firebase Console Settings.
- **OUTPUT:** `.gitignore` is updated to exclude `serviceAccountKey.json`. The key JSON file is safely placed in the `backend/` directory.
- **VERIFY:** Running `git status` verifies the JSON key is completely ignored by version control.

#### Task 5: Initialize Firebase Admin SDK
**Agent:** `backend-specialist` | **Skill:** `api-patterns`
- **Dependencies:** Task 4
- **INPUT:** `backend/requirements.txt` and Python server scripts.
- **OUTPUT:** `firebase-admin` added to requirements. Created `backend/app/core/firebase.py` to securely initialize the Admin SDK using the downloaded credential file.
- **VERIFY:** FastAPI server boots correctly, reads the credential, and connects to Firebase without throwing errors.
